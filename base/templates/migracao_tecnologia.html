Sprint 0 — Setup do Projeto (1-2 dias)
  • Criar projeto Django: django-admin startproject config .
  • Criar apps: base, usuarios, pacientes, medicos, agendas, enfermagem, financeiro
  • Configurar settings.py: MySQL backend, timezone America/Sao_Paulo, AUTH_USER_MODEL

  • Configurar Alpine.js (CDN no base.html)
  • Criar base.html com estrutura master (navbar, footer, toast)
  • Configurar django.contrib.messages para substituir o toast PHP

  • Criar AuditMixin (abstract model com uscad, usalt, dtcad, dtalt)
  • Criar TODOS os models Nível 0 (app core): Unidade, Convenio, Especialidade, TipoConsulta, TipoExame, TipoMaterial, FormaPagamento, StatusAgendamento, Material, LogAuditoria
  • Criar model customizado Usuario com AbstractUser (app accounts)
  • Criar model Permissao com todos os campos MNU_* e CAMPO_*, TAB_*
  • Implementar LoginView + LogoutView + middleware de sessão/permissões
  • Criar template de Login (login.html)
  • Criar template Home (home.html com botões condicionais por permissão)
  • Criar template de navegação (_navbar.html) com permissões
  • Rodar migrations e testar login
Sprint 2 — Tabelas de Configuração (3-5 dias)
  • Criar views genéricas (CreateView/UpdateView/ListView) para todas as tabelas Nível 0
  • Criar template genérico crud_list.html e crud_form.html (reutilizar para 9 CRUDs)
  • Telas: Convênios, Formas de Pagamento, Especialidades, Tipos de Consulta/Exame/Material, Materiais, Unidades
  • Criar CRUD de Usuários + tela de Permissões
  • Criar models Paciente, Medico, MedicoEspecialidade e rodar migrations
  • Implementar CRUD Paciente (list com paginação server-side, form completo)
  • Implementar CRUD Médico + gestão de Especialidades (M2M)
Sprint 3 — Pacientes e Médicos (2-3 dias)
  • Implementar API de busca de paciente (autocomplete → /api/pacientes/buscar/)
  • Implementar proxy CEP (/api/cep/<cep>/ chamando BrasilAPI)
  • Implementar Alpine.js para busca de CEP automática e máscara de campos
  • Implementar VacinaPaciente model e CRUD vinculado ao paciente
Sprint 4 — Agenda de Consultas (5-7 dias) ⚠️ Sprint mais complexa
  • Criar models MedicoAgendaConfig, Agenda, Consulta
  • Implementar lógica de geração automática de agenda (config → slots do dia)
  • Implementar tela de Agenda com Alpine.js (navegação dia, seleção médico)
  • Implementar modal de agendamento (busca paciente, tipo consulta, especialidade, convênio)
  • Implementar ações: agendar, bloquear, liberar, cancelar, trocar status
  • Implementar Configuração de Horários do Médico por Unidade
  • Implementar Agenda do Médico (visão filtrada pelo médico logado)
  • Implementar Procedimento model e CRUD vinculado ao paciente
Sprint 5 — Atendimento Médico (5-7 dias) ⚠️ Sprint mais complexa
  • Implementar tela de Atendimento Médico (carregar dados consulta + paciente + histórico)
  • Implementar auto-save de anamnese/conduta via AJAX (Alpine.js $fetch)
  • Criar model Receita e implementar CRUDs (medicamentos + exames) na tela de atendimento
  • Implementar finalização de atendimento (atualizar consulta + agenda)
  • Implementar templates de impressão de receita e pedido de exame
Sprint 6 — Enfermagem (3-5 dias)
  • Criar models Autorizacao, AgendaEnfermagem
  • Implementar tela de Autorizações (CRUD vinculado a procedimentos)
  • Implementar tela de Liberações (visão de autorizações prontas para agendar)
  • Implementar Agenda de Enfermagem (agendamento de procedimentos)
Sprint 7 — Financeiro + Finalização (3-5 dias)
  • Criar model Financeiro e implementar CRUD
  • Implementar lançamentos com filtros por data e tipo

  • Implementar sistema de Log de Auditoria (middleware ou signals Django)
  • Substituir backup_bd.php por management command dumpdata
  • Testes de integração end-to-end
  • Finalizar CRUD + Tela de Permissões de Usuários
  • Script de migração de dados do MySQL antigo → novo schema
